# AGENTS.md
항상 한국어로 답변해. 컴퓨터에 초보야.

이 파일은 `삼운몽: 세 개의 꿈` 프로젝트를 작업하는 Codex/AI 에이전트를 위한 프로젝트 지침이다. 게임 요소를 만들거나 수정하기 전에는 반드시 `DESIGN.md`를 먼저 읽고, 아래의 확정된 흐름과 충돌하지 않게 작업한다.

## 1. 프로젝트 정체성

- 이 프로젝트는 추리게임이다.
- 게임 제목은 `삼운몽: 세 개의 꿈`이다.
- 게임의 핵심은 꿈속 사건을 조사하고, 증거 수집과 용의자 심문을 통해 범인을 지목하는 것이다.
- 전체 게임 분위기는 특정 시대물로 고정하지 않고, 메인 화면처럼 어둡고 몽환적인 미스터리 추리 게임 톤을 기준으로 한다.
- 세 개의 꿈은 실제 시간대나 피해자의 기억이 아니라 플레이어의 환상이다.
- 플레이어는 꿈을 꾸는 인물이지만, 각 꿈 안에서는 특정 역할을 맡는다.
- 현재 우선 제작 범위는 조선시대 테마이다.

## 2. 작업 전 필수 확인

새 기능, 화면, 스토리, UI, 게임 규칙을 만들기 전에 다음을 확인한다.

1. `DESIGN.md`를 읽는다.
2. 현재 요청이 조선시대 테마에 속하는지 확인한다.
3. 구현 요청이면 Next.js 프로젝트 구조와 관례를 기준으로 설계한다.
4. 사용자가 이미 확정한 설정과 AI가 제안한 설정을 구분한다.
5. 용의자 관계, 범인, 알리바이, 핵심 반전은 사용자가 나중에 알려주기로 했으므로 임의로 확정하지 않는다.
6. 마법학교 테마와 우주 정거장 테마는 현재 구현 범위가 아니라 보류된 확장 후보로 다룬다.

## 3. 확정된 전체 흐름

게임은 아래 순서를 기본 흐름으로 가진다.

1. **메인 화면**
   - 게임 시작 시 가장 먼저 뜬다.
   - `NEW DREAM`, `CONTINUE`, `SETTINGS`, `EXIT` 같은 메뉴 구조를 사용할 수 있다.
   - 분위기는 어두운 꿈, 달밤, 미스터리, 범죄 현장 느낌을 우선한다.
   - 조선풍 분위기는 게임 전체 고정 톤이 아니라 조선시대 테마에서만 강하게 사용한다.

2. **튜토리얼**
   - 메인 화면 이후 튜토리얼이 나온다.
   - 튜토리얼에는 `SKIP` 버튼이 있어야 한다.
   - 튜토리얼은 꿈 세계관을 설명한다.
   - 핵심 설명: 꿈에서 깨려면 세 개의 사건을 해결해야 한다.

3. **꿈 선택 화면**
   - 플레이어에게 “어떤 꿈을 먼저 꾸시겠습니까?”라는 선택 상황을 제공한다.
   - 세 개의 테마 중 하나를 선택하는 구조이다.
   - 현재는 조선시대 테마만 먼저 제작하므로, 나머지 테마는 잠금, 준비 중, 또는 보류 상태로 둘 수 있다.

4. **꿈 진입 및 역할 인지**
   - 플레이어가 꿈을 선택하면 해당 꿈의 배경 설명이 나온다.
   - 이후 사용자가 꿈 안에서 맡게 되는 캐릭터/역할을 자연스럽게 알게 한다.
   - 조선시대 테마에서는 다른 캐릭터가 플레이어를 `사또님`처럼 부르며, 사용자가 자신이 꿈 안에서 사또 역할임을 인지하게 한다.
   - 이 역할 인지는 설명문만으로 처리하기보다, 캐릭터 대사와 상황을 통해 전달하는 것을 우선한다.

5. **조사와 심문 루프**
   - 핵심 플레이는 `사건 현장 <-> 취조실`을 오가며 진행된다.
   - 사건 현장에서는 포인트 앤 클릭 방식으로 증거를 수집한다.
   - 취조실에서는 대화형 AI 신문 방식으로 용의자를 심문한다.
   - 증거와 진술을 비교해 모순을 찾는다.

6. **범인 지목**
   - 플레이어는 수집한 단서와 심문 내용을 바탕으로 범인을 지목한다.
   - 제한시간은 없다.
   - 플레이어가 완전히 실패해서 막히는 구조가 아니라, 결국 진상에 도달해 범인을 지목할 수 있어야 한다.

7. **결과와 에필로그**
   - 범인 지목 후 결과가 나온다.
   - 마지막에는 꿈에서 깨는 에필로그 또는 아직 다른 사건이 남아 꿈에서 깨지 못하는 에필로그로 이어질 수 있다.
   - 세 개의 사건을 모두 해결해야 완전히 꿈에서 깬다는 구조를 유지한다.

## 4. 현재 조선시대 테마 기준

- 현재 만드는 첫 테마는 조선시대 추리 사건이다.
- 플레이어의 꿈속 역할은 사또로 가정한다. 단, 사용자가 다른 역할을 확정하면 그 결정을 따른다.
- 용의자는 총 4명이다.
- 용의자들의 관계, 이름, 알리바이, 숨기는 정보는 사용자가 이미 정해두었으므로 필요할 때 반드시 물어본다.
- 주요 화면 후보:
  - 메인 화면
  - 튜토리얼
  - 꿈 선택 화면
  - 조선시대 배경 설명 화면
  - 사또 역할 인지 장면
  - 사건 현장
  - 취조실
  - 증거 인벤토리
  - 추리/모순 정리 화면
  - 범인 지목 화면
  - 결과 화면
  - 에필로그

## 5. UI/UX 원칙

- 첫 화면은 랜딩 페이지처럼 설명만 있는 화면이 아니라, 실제 게임의 메인 화면처럼 보여야 한다.
- 튜토리얼은 반드시 건너뛸 수 있어야 한다.
- 사용자가 자신의 역할을 이해하는 장면은 몰입감을 해치지 않게 대사와 상황으로 보여준다.
- 사건 현장과 취조실 이동은 명확해야 한다.
- 증거 수집, 증거 확인, 용의자 심문, 범인 지목은 플레이어가 자연스럽게 반복할 수 있어야 한다.
- 제한시간 UI를 만들지 않는다.
- 실패 화면으로 플레이를 끊기보다, 추가 단서나 힌트를 통해 진상에 도달하게 한다.

## 6. Next.js 구현 원칙

- 새 구현은 Next.js를 기준으로 한다.
- 화면은 메인 화면, 튜토리얼, 꿈 선택, 사건 현장, 취조실, 증거 인벤토리, 범인 지목, 결과/에필로그처럼 흐름 단위로 나눈다.
- 게임 상태는 현재 꿈, 현재 장소, 수집 증거, 심문 진행도, 발견한 모순, 범인 지목 결과를 중심으로 설계한다.
- 조선시대 테마 리소스와 다른 테마 리소스는 섞지 않고 분리할 수 있게 구성한다.
- UI를 만들 때는 `DESIGN.md`의 비주얼 디자인 가이드를 우선한다.

## 7. 시각 톤

- 전체 게임 톤은 어둡고 몽환적인 미스터리 추리 게임 분위기를 기준으로 한다.
- 메인 화면처럼 꿈, 달빛, 범죄 현장, 비현실적인 공간감이 공통 분위기이다.
- 조선시대 테마에서는 어두운 밤, 안개, 등불, 젖은 돌길, 목재, 낡은 종이, 붉은 인장, 혈흔을 중심으로 한다.
- 조선시대 테마 UI는 현대적인 플랫 패널보다 고문서, 목패, 증거 표식, 봉인 인장, 낡은 지도 같은 물성을 우선한다.
- 단서 강조는 큰 화살표보다 빛 번짐, 작은 반짝임, 붉은 표시, 증거 번호 표식처럼 자연스러운 방식으로 한다.

## 8. 임의로 확정하지 말 것

다음 내용은 사용자가 직접 정하거나 나중에 알려주기로 한 영역이다. 에이전트가 임의로 확정하지 않는다.

- 용의자 4명의 이름
- 용의자들의 관계
- 범인
- 범행 동기
- 알리바이
- 피해자 신원
- 사건의 정확한 시간표
- 플레이어가 사또 외에 다른 역할을 맡는지 여부
- 세 개 테마의 정확한 이름과 순서

필요하면 사용자에게 짧게 질문하고, 답을 받은 뒤 `design.md` 또는 이 파일에 반영한다.

## 9. 문서 업데이트 규칙

- 사용자가 새로운 결정을 내리면 `design.md`를 우선 업데이트한다.
- 작업자 지침으로 반복해서 필요한 내용이면 `AGENTS.md`에도 반영한다.
- 출처를 구분한다.
  - 이미지에서 보이는 내용: 소스 사실
  - 사용자가 말한 내용: 사용자 결정
  - 에이전트가 제안한 내용: 제안 또는 보류 아이디어
- 사용자 결정이 있는 경우 에이전트 제안보다 항상 우선한다.
